blink, a 2D platforming game engine for Flash
Status: Ongoing (last updated June 2011)
 

Description:
This demo contains a little robot character that can roll around and jump. He starts out with a little guardian/helper thing that can transform into blocks; this ability is useful for getting around, but also for smashing the vacuum robots. The green guardian/helper thing has a friend...can you find him?

The demo shows the basic mechanics of the game engine; there are collisions between objects on the same level of depth, which are handled by a simple physics system. The engine can respond to keyboard and mouse input, and it can dynamically create and delete objects. A simple animation system is used to animate the characters depending on their current state. The engine is still very much under development, so don't be surprised if you find a few bugs.

 

  


blink level editor
Status: Ongoing (last updated June 2011)
 

Description:
This is the level editor that I created and used to build the demo above. Much like the engine itself, this is also a work in progress. You first select the size of the level you want to create (I suggest using the default tile size or 150); then you select a tile that you want to use to 'paint' the grid with from the tab on the left. You can also add other preloaded or imported objects into the level using the other tab on the left.

The Toolbar Tab contains these Tools from top to bottom: Tile Paintbrush, Eraser, Workspace Grab, Depth Test, Zoom In, Zoom Out, Select, Grab, and Rotate.

The Level of Depth Tab on the bottom starts out with the default depth (0) selected and visible. Clicking other depths will generate more grids behind or in front of the default depth. You can interact with these just like the default depth grid; you can also hide or lock various depth so that they don't get in your way.

If you want to try Importing images, here are some that are loaded on the server that you can import: smallgrass.png, smallcloud.png, rocks.png, pondtop.png, grass.png, cloud.png, blackPillar.png, blackBlock_small.png, blackBlock.png, and TiltedBlock.png. You can interact with the bounding box by grabbing it or its edges, or you can input values directly into the input fields. An imported image can only be made into a tile if its Collision Box has the same width and height and 0 rotation.
